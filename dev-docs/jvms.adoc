# Gradle and JVMs

Solr uses Gradle's Java toolchain support to separate the JDK running Gradle from the JDKs used for compilation and testing.

## Requirements

- **To run Gradle**: Java 17 or later
- **Main code compilation**: Java 21
- **SolrJ modules compilation**: Java 17 (or Java 21 with language level 17)
- **Tests**: Java 21 for main code, Java 17 for SolrJ modules

## JDK Discovery

Gradle automatically discovers installed JDKs from:

- `JAVA_HOME` environment variable
- Common installation locations (platform-specific):
  - macOS: `/Library/Java/JavaVirtualMachines/`
  - Linux: `/usr/lib/jvm/`, `/usr/java/`
  - Windows: `C:\Program Files\Java\`

If Gradle cannot find your JDKs, you can manually specify them in `~/.gradle/gradle.properties`:

----
org.gradle.java.installations.paths=/path/to/jdk17,/path/to/jdk21
----

## Verification

List detected JDKs:
----
./gradlew -q javaToolchains
----

Verify which JDK compiles specific modules:
----
./gradlew :solr:core:compileJava --info | grep -i "toolchain"
./gradlew :solr:solrj:compileJava --info | grep -i "toolchain"
----

## Running Gradle with Different Java Versions

You can run Gradle itself with any Java 17+ installation:
----
JAVA_HOME=/path/to/jdk17 ./gradlew build
----

Gradle will still use the configured toolchains (Java 21/17) for compilation and testing.

See https://docs.gradle.org/current/userguide/toolchains.html[Gradle's toolchain documentation] for more information.
